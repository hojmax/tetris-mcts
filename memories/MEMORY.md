- 2026-05-02: `run_search` now seeds `q_bounds` from the carried tree on the first simulation by walking visited descendants and taking min/max of their `mean_value()` (helper `seed_q_bounds_from_tree` in `tetris_core/src/search/search.rs`). Fresh roots still start at `(0.0, 0.0)` because they have no visits. This keeps reused-subtree PUCT normalization meaningful from sim 0 instead of collapsing every visited child to 0.5 until enough new rollouts widen bounds.
- 2026-05-02: Training hot-loop GPU/MPS sync hygiene: keep the per-step path off `.item()`/scalar `.any()` on device tensors. Fixed sources to watch in future: `apply_action_mask` had a `(valid_counts == 0).any()` debug guard (now extracted to opt-in `validate_action_masks_have_valid_rows`), `policy_mirroring.maybe_mirror_training_tensors` used `torch.rand((), device=boards.device).item()` (now CPU-tensor), and `torch.nn.utils.clip_grad_norm_` always returned a device tensor (now skipped when `grad_clip_norm <= 0`, logging `nan`). Treat `optimizer.grad_clip_norm <= 0` as "disabled" because PyTorch silently zeroes gradients at `max_norm=0.0`.
- 2026-05-02: `torch.compile` auto-gated off on MPS in `Trainer.__init__` via `_effective_use_torch_compile` (which also feeds `AdamW(foreach=...)`). Apple Silicon eager is faster than the compiled path for this network; CUDA still compiles when `optimizer.use_torch_compile=true`.

- 2026-03-23: Resume + `mirror_replay_on_accelerator=True` can OOM at `step=0` when the replay buffer is already full (for example `4,000,000` examples loaded from `training_data.npz`). `Trainer._load_replay_mirror()` currently calls `generator.replay_buffer_snapshot()`, converts the entire snapshot to a device batch via `_to_training_device(...)`, and only then allocates `CircularReplayMirror`. Because Rust exports snapshot arrays as dense `float32` NumPy arrays for boards/masks/policy, the transient peak is `full float32 device batch (~7008 bytes/example)` plus the packed mirror allocation (`~4203 bytes/example`), so a 4M-example resume needs roughly `26.11 GiB + 15.66 GiB` before the temporary batch can die. On 32 GB GPUs this fails during mirror bootstrap even though the steady-state packed mirror alone fits. Fresh runs usually avoid this because the first snapshot is small and later updates use incremental deltas.
